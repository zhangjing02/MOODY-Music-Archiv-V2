# 🚀 纯云端上传架构设计方案

## 📋 需求分析

### 核心需求
1. ✅ 用户可以在任何地方打开管理后台
2. ✅ 上传音乐文件，直接落盘到 Cloudflare R2
3. ✅ 元数据直接写入 Cloudflare D1
4. ✅ 实时反馈上传状态和错误信息
5. ✅ 不依赖本地文件系统

### 当前问题
❌ 文件先保存到本地 `storage/music/`
❌ 元数据写入本地 SQLite（前端读不到）
❌ R2 上传在 goroutine 中异步执行，失败无反馈
❌ 需要 R2 环境变量

---

## 🎯 推荐方案：Cloudflare Worker 直传

### 架构设计

```
前端管理后台
    ↓
    ↓ (FormData + 元数据)
    ↓
Cloudflare Worker API
    ↓
    ├─→ 1. 接收文件流
    ├─→ 2. 直接上传到 R2 (使用 PutObject)
    ├─→ 3. 从 R2 读取文件元数据 (ID3 tag)
    ├─→ 4. 写入 Cloudflare D1
    └─→ 5. 返回详细结果给前端
```

### 优势
✅ **完全云端化**：不依赖本地文件系统
✅ **实时反馈**：同步执行，有错误立即返回
✅ **数据一致**：直接写入 D1，前端立即可见
✅ **无需环境变量**：R2 配置在 Worker 中
✅ **全球可用**：通过 Cloudflare CDN，任何地方都能访问

## 📝 实现方案

### 方案 A：使用 Cloudflare Worker R2 Bindings（推荐）

#### 原理
Cloudflare Worker 支持直接绑定 R2 Bucket，无需 AWS SDK

#### 实现步骤

**1. 在 Cloudflare Worker 中添加上传 API**

```typescript
// cloudflare-worker/src/index.ts

// 上传单个文件到 R2 并写入 D1
app.post('/api/admin/upload', async (c) => {
  try {
    const formData = await c.req.formData()
    const file = formData.get('file') as File
    const artistOverride = formData.get('artistOverride') as string
    const albumOverride = formData.get('albumOverride') as string

    if (!file) {
      return c.json({ code: 400, message: '未检测到文件' }, 400)
    }

    // 1. 生成对象键（使用 ID3 tag 或覆盖参数）
    const arrayBuffer = await file.arrayBuffer()
    const metadata = await extractMetadata(arrayBuffer, file.name)

    const artist = artistOverride || metadata.artist || '未知艺术家'
    const album = albumOverride || metadata.album || '未知专辑'
    const title = metadata.title || file.name

    // 2. 上传到 R2
    const objectKey = `music/${artist}/${album}/${file.name}`
    await c.env.BUCKET.put(objectKey, arrayBuffer, {
      httpMetadata: {
        contentType: file.type
      }
    })

    // 3. 查找或创建艺人、专辑记录
    let artistId = await findOrCreateArtist(c, artist)
    let albumId = await findOrCreateAlbum(c, album, artistId, metadata.year)

    // 4. 插入歌曲记录
    const songResult = await c.env.DB.prepare(
      `INSERT INTO songs (title, file_path, lrc_path, album_id, storage_id, track_index)
       VALUES (?, ?, ?, ?, ?, ?)`
    ).bind(
      title,
      objectKey,
      null,
      albumId,
      'primary',
      metadata.trackNumber || 0
    ).run()

    return c.json({
      code: 200,
      message: '上传成功',
      data: {
        songId: songResult.meta.last_row_id,
        title: title,
        artist: artist,
        album: album,
        objectKey: objectKey
      }
    })
  } catch (error: any) {
    return c.json({
      code: 500,
      message: '上传失败: ' + error.message
    }, 500)
  }
})

// 辅助函数：从 MP3 提取元数据

  // 使用纯 JavaScript 或调用外部 API 解析 ID3 tag
  // 这里简化处理，实际需要集成 id3 库
  return {
    title: '',
    artist: '',
    album: '',
    year: '',
    trackNumber: 0
  }

// 辅助函数：查找或创建艺人
async function findOrCreateArtist(c: any, name: string): Promise<number> {
  const result = await c.env.DB.prepare(
    'SELECT id FROM artists WHERE name = ?'
  ).bind(name).first()

  if (result) {
    return result.id
  }

  const insertResult = await c.env.DB.prepare(
    'INSERT INTO artists (name) VALUES (?)'
  ).bind(name).run()

  return insertResult.meta.last_row_id
}

// 辅助函数：查找或创建专辑
async function findOrCreateAlbum(c: any, title: string, artistId: number, year: string): Promise<number> {
  const result = await c.env.DB.prepare(
    'SELECT id FROM albums WHERE title = ? AND artist_id = ?'
  ).bind(title, artistId).first()

  if (result) {
    return result.id
  }

  const insertResult = await c.env.DB.prepare(
    'INSERT INTO albums (title, artist_id, release_date) VALUES (?, ?, ?)'
  ).bind(title, artistId, year || null).run()

  return insertResult.meta.last_row_id
}
```

```javascript
// frontend/admin/admin.js

async function uploadToWorker(file, artist, album) {
  const formData = new FormData()
  formData.append('file', file)
  if (artist) formData.append('artistOverride', artist)
  if (album) formData.append('albumOverride', album)

  const response = await fetch(
    'https://moody-worker.changgepd.workers.dev/api/admin/upload',
    {
      method: 'POST',
      body: formData
    }
  )

  const result = await response.json()

  if (result.code === 200) {
    console.log('✅ 上传成功:', result.data)
    return result.data
  } else {
    console.error('❌ 上传失败:', result.message)
    throw new Error(result.message)
  }
}
```

### 方案 B：使用 Presigned URL（更复杂但更灵活）

如果遇到 Worker 请求大小限制（Cloudflare Workers 有 100MB 请求限制），可以使用 Presigned URL：

#### 实现步骤

**1. 创建获取 Presigned URL API**

```typescript

  const artist = c.req.query('artist')
  const album = c.req.query('album')

  const objectKey = `music/${artist}/${album}/${filename}`

  // 这是一个限制，所以推荐方案 A
})
```

**2. 前端直接上传到 R2**

```javascript
async function uploadDirectToR2(file, artist, album) {
  // 1. 获取上传 URL
  const urlResponse = await fetch(
    `https://moody-worker.changgepd.workers.dev/api/admin/upload-url?filename=${file.name}&artist=${artist}&album=${album}`
  )
  const { uploadUrl } = await urlResponse.json()

  // 2. 直接上传到 R2
  await fetch(uploadUrl, {
    method: 'PUT',
    body: file
  })

  // 3. 通知后端完成入库
  await fetch('https://moody-worker.changgepd.workers.dev/api/admin/upload-complete', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      filename: file.name,
      artist: artist,
      album: album
    })
  })
}
```

## ⚠️ 方案对比

| 特性 | 方案 A（Worker 直传） | 方案 B（Presigned URL） |
|------|---------------------|----------------------|
| **复杂度** | ✅ 简单 | ⚠️ 复杂 |
| **实时反馈** | ✅ 同步执行 | ⚠️ 需要两次请求 |
| **错误处理** | ✅ 立即返回错误 | ⚠️ 需要额外的回调 |
| **文件大小限制** | ⚠️ 100MB（Worker 限制）| ✅ 无限制（R2 直传） |
| **依赖** | ✅ 无需额外 SDK | ⚠️ 需要 AWS SDK |
| **R2 支持** | ✅ 原生支持 | ⚠️ 需要兼容性处理 |

**推荐**：方案 A（Worker 直传），除非文件超过 100MB

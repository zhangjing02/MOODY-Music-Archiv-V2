# 性能优化方案

## 数据结构

```
分类 (CATEGORIES)
├── 全部
├── 华语
├── 港台
├── 欧美
└── ... (共10个分类)

歌手 (MOCK_DB)
├── id: 唯一标识
├── name: 歌手名
├── group: 首字母 (A-Z)
├── category: 所属分类
├── avatar: 头像图片URL
└── albums: 专辑列表
    ├── title: 专辑名
    ├── year: 发行年份
    ├── cover: 封面图片URL
    └── songs: 歌曲列表 (10-20首)

预估数据量：
- 歌手：100-1000人
- 每位歌手：5-20张专辑
- 每张专辑：10-20首歌
- 总图片数：500-20,000张
```

## 页面呈现逻辑

### 1. 侧边栏（固定左侧）
- 分类标签（水平滚动，10个）
- 艺术家列表（按首字母分组，垂直滚动）
- 字母索引条（快速跳转）

### 2. 主内容区（右侧）
- 专辑封面 (400x400)
- 专辑信息
- 歌词显示区
- 专辑标签（水平滚动）
- 歌曲列表（表格，10-20行）

### 3. 播放器（固定底部）
- 进度条
- 控制按钮
- 专辑封面缩略图

## 懒加载策略

### 🎯 第一优先级：关键渲染路径

**立即加载（阻塞渲染）：**
```
├── HTML 结构
├── CSS 样式
├── 核心框架代码 (app.js, player.js 核心部分)
└── 基础数据 (歌手列表元数据，不含图片)
```

**延迟加载（非阻塞）：**
```
├── 歌词数据 (LYRICS_LRC_DB)
├── 设置模块 (storage.js)
└── 图片资源（全部）
```

### 🖼️ 第二优先级：图片懒加载

#### A. 艺术家头像（60x60）
```javascript
策略：
├── 初始：只加载首屏可见的前 20 个
├── 滚动：使用 Intersection Observer 触发加载
├── 占位：使用 UI Avatars API 作为默认占位图
└── 降级：加载失败时显示首字母缩写

优先级：低（列表项，非核心内容）
```

#### B. 专辑封面（400x400）
```javascript
策略：
├── 首屏：当前选中专辑的封面（立即加载）
├── 预加载：当前艺术家的其他专辑封面（优先级：高）
├── 懒加载：滚动到视野内时加载
├── 缩略图：先加载 100x100，点击放大后加载 600x600
└── 缓存：已加载的封面永久缓存

优先级：中（主视觉，但可接受渐进显示）
```

#### C. 播放器封面（100x100）
```javascript
策略：
├── 立即加载：当前播放歌曲的封面
├── 预加载：播放列表中的下一首
└── 缓存：永久缓存

优先级：高（用户体验核心）
```

### 📜 第三优先级：虚拟滚动

**艺术家列表虚拟滚动：**
```javascript
配置：
├── 可见区域高度：约 600px
├── 每项高度：72px
├── 可见数量：约 8-10 个
├── 缓冲区：上下各额外渲染 5 个
└── 总渲染数：约 20 个（实际可能有几百个）

实现方案：
├── 监听滚动事件计算可见范围
├── 只渲染可见区域 + 缓冲区的元素
├── 使用 transform 定位元素
└── 维护虚拟滚动位置
```

### 🚀 第四优先级：预加载策略

#### A. 智能预加载
```javascript
// 选择艺术家后
预加载内容：
├── 当前艺术家的所有专辑封面
├── 第一首歌的歌词
└── 第一首歌的音频（如果有本地文件）

// 播放歌曲时
预加载内容：
├── 下一首歌的歌词
├── 下一首歌的音频
└── 下一首的专辑封面
```

#### B. 空闲时预加载
```javascript
使用 requestIdleCallback：
├── 预加载相邻首字母的艺术家头像
├── 预加载热门歌手的专辑封面
└── 预缓存常用歌词
```

### 📦 第五优先级：数据分页（面向未来 API）

```javascript
// 歌手列表分页
GET /api/artists?page=1&limit=50&category=全部

// 专辑列表分页
GET /api/artists/:id/albums?page=1&limit=20

// 歌曲列表分页
GET /api/albums/:id/songs?page=1&limit=20
```

### ⚡ 第六优先级：代码分割

```javascript
// 核心代码（立即加载）
├── app.js (基础渲染逻辑)
├── player.js (播放器核心)
└── data.js (歌手元数据)

// 非核心代码（延迟加载）
├── lyrics.js (歌词库，约 500KB+)
├── storage.js (存储管理)
└── utils.js (工具函数)
```

## 性能指标

### 目标
- **首屏加载时间 (FCP)**: < 1.5s
- **可交互时间 (TTI)**: < 3s
- **内存占用**: < 50MB（初始）
- **帧率**: 60fps（滚动时）

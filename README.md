# MOODY Music Archive 🎵

一个极简、优雅且功能强大的私有音乐归档与展示系统。支持多机协同同步、自动化资产管理以及 Cloudflare R2 透明代理。

## 1. 项目结构与目录 (The Anatomy)

项目采用前后端分离架构，配合 AI Agent 技能体系进行高效运维。

```text
.
├── frontend/                # [Vite + Vue3] 前端展示层
├── backend/                 # [Go] 后端逻辑层 (主服务: 8080, 管理: 8082)
│   ├── internal/            # 核心业务逻辑 (存储代理、数据库访问)
│   └── main.go              # 服务入口
├── storage/                 # [Local] 本机音频与封面物理存储 (建议由 .env 指定)
│   └── db/                  # [SQLite] 数据库文件目录
├── tools/                   # [Scripts] 独立运维脚本
│   └── migrate/             # R2 增量同步系统 (支持 Manifest 去重)
└── .agent/                  # [AI Agent] 核心能力集成
    ├── skills/              # 结构化技能 (资产治理、大满贯录入等)
    └── workflows/           # 自动化工作流 (新歌入库、一键同步)
```

## 2. 项目部署方案 (Deployment Strategy)

### 2.1 环境变量配置 (`.env`)
在 `backend/` 和 `tools/migrate/` 目录下根据 `.env.example` 创建 `.env` 文件：

| 变量名 | 说明 | 示例 |
| :--- | :--- | :--- |
| `MOODY_STORAGE_PATH` | 本地资源物理路径 | `E:\Html-work\storage` |
| `MOODY_PORT` | 主播放服务端口 | `8080` |
| `MOODY_ADMIN_PORT` | 管理服务端口 | `8082` |
| `R2_BUCKET_NAME` | Cloudflare R2 Bucket 名 | `moody-music` |
| `R2_ACCOUNT_ID` | Cloudflare 账户 ID | `(从控制台获取)` |
| `R2_ACCESS_KEY_ID` | R2 API 访问密钥 ID | `(从控制台获取)` |
| `R2_SECRET_ACCESS_KEY` | R2 API 访问密钥 Secret | `(从控制台获取)` |

### 2.2 云端部署 (ClawCloud Run Deployment)
V2 版本已针对 **ClawCloud Run** 进行深度适配，支持“GitHub 推送即部署”：

1. **关联仓库**：在 [ClawCloud Run](https://claw.cloud) 面板中关联此 GitHub 仓库。
2. **自动构建**：系统会自动识别根目录下的 `Dockerfile` 并执行多阶段构建。
3. **环境注入**：在面板中配置前述环境变量（R2 凭证、存储路径等）。
4. **持久化存储**：建议在 ClawCloud 挂载 **Persistent Storage** 至 `/app/storage`，确保数据库 `.db` 文件不丢失。

### 2.3 零感架构 (Invisible R2 Architecture)
V2 核心升级：
- **智能调度**：支持通过 `storage_id` 在多个 R2 账户间无缝调度资源。
- **透明代理**：通过 `StorageProxyHandler` 实现 S3 优先读取。在 ClawCloud 环境下，即使容器磁盘空间有限，也能通过代理直接拉取云端 7 万+ 资产。

## 3. 运维手段与工具箱 (Ops & Maintenance)

### 3.1 核心 API 接口名录
详细参数请参考 [API 接口文档](api/API.md)。

| 接口名 | 端口 | 说明 |
| :--- | :--- | :--- |
| `/api/skeleton` | 8080 | 拉取全站歌手/专辑/歌曲层级树 |
| `/api/songs` | 8080 | 歌曲详情与播放路径获取 |
| `/api/admin/governance` | 8082 | **核心治理**：同步、清理、去重、歌词绑定 |
| `/api/admin/scrub` | 8082 | 强力对齐：剔除幽灵脏名录 + 物理清扫冗余 |
| `/api/sync/manifest` | 8082 | 管理 R2 资产清单 (Manifest) |

### 3.2 Agent 核心技能 (Skills)
借助 AI Agent，您可以执行以下高阶指令：
- **`storage_sync`**：增量同步本地 `storage` 至 Cloudflare R2，支持多机校验。
- **`file_management`**：自动处理新上传的歌曲（ID化、lrc 匹配、生成 _contents.txt）。
- **`music`**：通过 iTunes API 自动录入艺术家的完整专辑骨架。

### 3.3 自动化工作流 (Workflows)
1.  **`/music-align-sync`**：一键完成「新歌 ID 化 -> 数据库入库 -> R2 同步」的全流程。
2.  **`/storage-sync`**：快速在不同设备间对齐 R2 资产状态。

### 3.4 调试与维护
- **数据库修复**：底层使用 SQLite，文件位于 `storage/db/moody.db`。
- **ID 化命名意义**：封面 `c_ID.jpg` 与歌曲 `s_ID.mp3` 的命名方案规避了多平台下的编码乱码风险，提升了寻址效率。
- **R2 同步排障与加速**：若出现文件不一致，尝试删除本地 `manifest.json` 后重新运行。为提升效率，推荐使用**科学同步模式**：
  ```bash
  node migrate.mjs --target music/歌手名  # 定向同步
  node migrate.mjs --days 3              # 仅同步近 3 天变更
  ```
